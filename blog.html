Discuss the scope of var, let, and const

in javascript user can declared three variable keywords such as var, let, const the scop of var, let, const are discuss below.

VAR: var variable is a global variable that can be called from outside the function. It means variables defined outside the function can be accessed globally

LET: A let variable is a block type variable, which cannot be called outside a specific block

CONST: When users declare a const variable, they need to initialize it, otherwise, it returns an error. The user cannot update the const variable once it is declared.

Tell us the use cases of null and undefined?

Null: The primitive type null indicates the intentional absence of a value. Therefore, in JavaScript, programmers often assign null to a variable to denote that the variable is declared and initialized and currently doesn’t hold a value. Still, a value can be expected in the future


Undefine: undefined is a primitive value automatically assigned by JavaScript to indicate the unintentional absence of any object value
